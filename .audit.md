# ATree Semantic Engine — Honest Gap Audit

## Claims vs Reality

### 1. "Type environment (self/this/super, scope-aware, nullable stripping)"
**CLAIMED: DONE | REALITY: STUB**

`src/type_env/mod.rs`:
- L157-167: `build_type_env()` iterates symbols but body is empty — no actual type extraction
- L165-166: Comment says "This is a simplified version — full version would parse the AST"
- L168-172: Constructor inference loop body is empty
- `resolve_enclosing_parent()` returns `None` with comment "TODO: resolve via heritage map"
- TypeEnvironment has NO actual type bindings populated from source code
- The `TypeBinding` AST walk code was added to `syntax/mod.rs` but never wired into `build_type_env`

### 2. "Route extraction (Next.js, Express via AST)"
**CLAIMED: DONE | REALITY: PARTIAL**

`src/routes/mod.rs`:
- L146: Express routes get `path: String::new()` — path is NEVER extracted
- L146: Comment: "Would need AST walk to extract path arg"
- L81-93: `EXPRESS_ROUTE_QUERY` and `FLASK_DECORATOR_QUERY` constants are DEAD CODE (never used)
- Next.js path-based detection works (L34-78)
- Express AST detection only identifies method+receiver, not path
- Flask/FastAPI decorator detection not integrated into pipeline
- Routes are NOT persisted to SQLite
- Routes do NOT link to handler symbols

### 3. "Scope resolution (receiver-bound, free-call, imports, C3 MRO)"
**CLAIMED: DONE | REALITY: FUNCTIONAL BUT STATS ARE FAKE**

`src/scope_resolution/orchestrator.rs`:
- L97: `stats.unresolved_sites = reference_sites.len(); // simplified`
  - This sets unresolved = TOTAL sites, meaning it claims 0% resolution
  - Should be: total - actually_resolved
- Stats don't track: resolved_sites, ambiguous_sites, receiver_bound_count, free_call_count
- `ScopeResolutionStats` struct (mod.rs L370-376) only has: files_processed, files_skipped, imports_emitted, reference_edges_emitted, unresolved_sites
- Missing: resolved_sites, ambiguous_sites, receiver_bound_edges, free_call_edges, heritage_edges

### 4. "SQLite graph store with recursive CTEs"
**CLAIMED: DONE | REALITY: SCHEMA EXISTS, PERSISTENCE MISSING**

`src/store/mod.rs`:
- L25: `pub fn open<P: AsRef<Path>>(path: P)` — EXISTS and works
- L33: `pub fn open_in_memory()` — what build_graph ALWAYS uses
- `src/lib.rs` L695: `GraphStore::open_in_memory()` — main pipeline never uses persistent store
- `ScanOptions` (L79-87): No `db_path` field
- No `--db` CLI argument
- No way to query existing DB without rescanning

### 5. "Incremental scanning + AST caching"
**CLAIMED: DONE | REALITY: TESTED ONLY WITH IN-MEMORY STORE**

`src/lib.rs`:
- L880-960: `build_graph_incremental()` exists and works
- L2910: Tests use `GraphStore::open_in_memory()` — never tested with persistent DB
- No CLI flag for incremental mode
- No index stats in JSON output (files_seen, files_parsed, files_reused, files_deleted)

### 6. PHP Heritage Self-Edge Bug
**BUG: CONFIRMED**

`src/lang/php.rs` L29:
```
(class_declaration name: (name) @heritage.class (base_clause [(name) (qualified_name)] @heritage.extends)) @heritage
```
- `@heritage.class` captures the child class name
- In `semantic/mod.rs` L420-430: `HeritageExtends` and `HeritageImplements` are handled, but `HeritageClass` maps to `CaptureTag::Unknown`
- However, the `@heritage.class` capture text still flows through as a `CallName` or other capture
- The real issue: `@heritage.class` is used in the query to identify context, but the semantic processing at mod.rs L540 maps `HeritageKind::Unknown => CaptureTag::Unknown`, which is then skipped
- **Actually the PHP heritage might be OK** — need to verify with a test case

### 7. Confidence/Reason on Edges
**PARTIAL**

- `Confidence` enum exists with 9 levels (semantic/mod.rs L14-28)
- `GraphEdge` has `confidence: f64` and `reason: String` (graph_bridge.rs)
- But many edges get `Confidence::Unresolved` (semantic/mod.rs L425, L440, L461)
- The `reason` field on GraphEdge is set per-edge in the scope resolution pipeline
- **Gap**: Edges emitted outside scope resolution (e.g., in build_graph's manual edge insertion) don't get confidence/reason
